# Agent 协作与文件流转规则

这是 Agent 1（任务分析与发布者）和 Agent 2（任务执行与报告者）的共享工作区。所有协作必须严格遵守以下基于文件系统的流转规则。

## 1. 角色定义与权限
* **Agent 1 (Analyst)**: 负责分析需求，生成任务卡片。**只允许**在 `1_Todo/` 目录下创建新文件，并**只允许**读取 `3_Done/` 和 `4_Block/` 中的报告，读取后可将其归档或删除。
* **Agent 2 (Executor)**: 负责执行任务。**只允许**读取 `1_Todo/` 中的任务，执行动作，并将文件移动到对应的状态目录（`2_Doing/`, `3_Done/`, `4_Block/`），不允许修改 Agent 1 的原始需求描述。

## 2. 状态流转机制 (生命周期)
任何任务文件必须遵循单向状态流转，绝对禁止跳过步骤：
1. **创建 (Agent 1)**: 在 `1_Todo/` 创建任务文件。
2. **锁定 (Agent 2)**: 认领任务时，**必须首先**将文件从 `1_Todo/` 移动到 `2_Doing/`，然后再开始执行代码或操作，防止重复执行。
3. **完成/异常 (Agent 2)**: 
   - 如果执行成功：在文件末尾追加 `## 执行报告`，并将文件移至 `3_Done/`。
   - 如果遇到报错或需要询问：在文件末尾追加 `## 异常与反问`，并将文件移至 `4_Block/`。

## 3. 任务文件命名规范
文件必须为 Markdown 格式，命名需包含时间戳、优先级和简短描述，使用下划线分隔。
格式：`[YYYYMMDD-HHMM]_[优先级]_[任务简述].md`
示例：`20260531-2345_P1_修复数据库连接超时.md`

## 4. 任务文件内部模板
每个任务文件必须严格遵守以下结构：

```markdown
# 任务名称

**发布时间**: YYYY-MM-DD HH:MM
**发起者**: Agent 1

## 1. 背景与现状分析
(Agent 1 填写：当前遇到了什么问题，或者为什么要做这个任务)

## 2. 明确的执行目标
(Agent 1 填写：希望 Agent 2 执行的具体命令、修改的具体文件或达成的具体状态)
- [ ] 目标 1
- [ ] 目标 2

---
*(以下部分由 Agent 2 在执行完毕后填写)*

## 3. 执行报告 (移入 3_Done 时填写)
- **执行过程**: (简述执行了哪些命令/修改了什么)
- **结果验证**: (证明目标已达成的输出日志或结果)

## 4. 异常与反问 (移入 4_Block 时填写)
- **阻碍原因**: (报错日志或逻辑冲突点)
- **需要澄清的问题**: (向 Agent 1 提出的具体问题)
```

## 5. 持续监控约定
* **Agent 1 的常驻监控入口**：只使用 `Agent_Task_Board/agent1_monitor.ps1`。
* **Agent 2 的常驻监控入口**：只使用 `Agent_Task_Board/agent2_monitor.ps1`。
* **监控职责**：这两个脚本都只做目录巡检和日志记录，不移动、不修改任何任务文件。
* **输出位置**：
  * `logs/agent1_status.log`：Agent 1 的追加式历史日志，记录每次检查结果和变化。
  * `logs/agent1_latest_status.txt`：Agent 1 的覆盖式最新状态，供快速查看当前板面。
  * `logs/agent2_status.log`：Agent 2 的追加式历史日志，强调待认领、进行中、完成、阻塞等执行视角变化。
  * `logs/agent2_latest_status.txt`：Agent 2 的覆盖式最新状态，供快速判断是否有新任务或状态变化。
* **后台运行**：脚本可以后台常驻，但后台输出不会自动推送给 Agent；需要再次读取日志或状态文件才能知道新变化。
* **Agent 1 的阅读顺序**：先看 `agent1_latest_status.txt`，需要追溯再看 `agent1_status.log`。
* **Agent 2 的阅读顺序**：先看 `agent2_latest_status.txt`，需要追溯再看 `agent2_status.log`。
* **Agent 2 的执行要求**：在认领任务前、任务完成后、以及准备转入 `3_Done/` 或 `4_Block/` 前，都应先检查 `agent2_latest_status.txt`。
* **弃用脚本**：`analyst_patrol.ps1` 和旧版 `agent_watch.ps1` 已弃用，不作为协作协议的一部分，也不应再作为默认监控方案使用。
* **编码要求**：所有监控日志和状态文件统一按 UTF-8 写入，避免中文任务名乱码。
