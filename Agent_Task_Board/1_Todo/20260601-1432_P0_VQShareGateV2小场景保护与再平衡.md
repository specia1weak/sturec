# VQShareGateV2 小场景保护与再平衡

**发布时间**: 2026-06-01 14:32
**发起者**: Agent 1

## 1. 背景与现状分析
`vq_share_gate_v1` 已经证明“轻量门控残差增强”是有效方向：

- best `overall.auc` 已达到 `0.786620`
- 相比 plain `vq_share` 有稳定提升
- 比 `ple_balanced_v3` 的单轮高点还略高

但它的不足也已经清楚暴露：

- epoch1 -> epoch2 的收益主要落在大场景
- `domain4(tab6)` 明确被 `specific` 过度修正拖累
- `domain2(tab2)` 虽然样本更少，但 counterfactual 显示它仍受益于 `specific`
- gate 最终更偏向 `specific`，不是“共享保护 + 局部修正”的平衡态

因此下一步研究重点不是继续加大门控复杂度，也不是一刀切压低所有低频场景的 residual，而是做“按场景风险差异”的再平衡结构。

## 2. 明确的执行目标
- [ ] 在 `src/custom_models` 基础上实现 `vq_share_gate_v2` 或同等命名版本。
- [ ] 让 gate 对高风险场景，优先是 `domain4(tab6)`，有更强的 shared 保护，避免 specific 放大噪声拖累。
- [ ] 避免把 `domain2(tab2)` 这类虽然低频但仍受益于 residual 的场景一并压死。
- [ ] 优先考虑轻量方案，例如：
  - domain-aware shared prior
  - risk-aware gate bias
  - 对 `domain4(tab6)` 这类高风险场景的 shared floor 提升
  - 利用 `residual_norm` / `quantized_cos` 做 sample-level 保守路由
- [ ] 保留现有 observatory 接口，重点新增小场景相关的门控诊断。
- [ ] 跑至少 1 轮 `seed=2024` 或 `seed=2025` 的 2 epoch 快速验证，先看小场景是否回升。
