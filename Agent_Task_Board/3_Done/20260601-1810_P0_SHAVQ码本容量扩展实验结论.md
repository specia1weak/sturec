## 5. 结论

1. 在当前结构下，单纯把码本容量从 `64 -> 128` 没有带来收益，反而整体和各场景指标均有回落。
2. 这说明当前主要瓶颈仍然不是“码本容量不足”，而是“共享与专属信息如何分层、如何更新”的结构问题。
3. 因此继续盲增容量的优先级应低于结构性改造，例如：
   - 共享第一层码本 + 专属第二层码本
   - 共享码本只建模通用信息，专属码本只建模剩余差异

## 6. 后续候选方向

1. 两层码本：
   - 第一层共享码本量化底层通用部分
   - 第二层专属码本量化共享残差
   - 剩余残差再交给专家网络
2. 若两层码本方向仍无明显收益，再回头评估是否需要：
   - 更大的 shared codebook
   - 或底层特征差异化增强（类似 PEPNet）
