## 3. 执行报告 (移入 3_Done 时填写)
- **执行过程**:
  1. 复核 `curriculum_ssim` 接管后的 4 份日志，确认该路线已从“明显退化”修到“接近基线”，但仍未超过原始 `ssim`。
  2. 对 `shavq_v4` 读取 `observatory_shavq_v4_stdout.log` 与配套图像，重点核查：
     - 两级码本是否健康
     - shared/specific 的方差与有效秩
     - context/innovation 两条专属路是否分工
     - shared logits 是否真正参与最终决策
  3. 对 `vq_share` 读取 `test.log` 中真实的 `vq_share` 运行段，以及 `observatory/vq_share/*` 图像，重点核查：
     - codebook 是否充分使用
     - shared route 是否稳定
     - shared logit 是否具有持续话语权
  4. 横向扫描 `workspace/kuairand-rand/*.log`，确认当前单轮分数较高的候选，并补查 `ple_balanced_v3` 结构说明。
- **结果验证**:
  1. `curriculum_ssim` 当前结论：
     - `workspace/kuairand-rand/curriculum_ssim_seed2024.log` best `overall.auc = 0.785348`
     - `workspace/kuairand-rand/curriculum_ssim_seed2024_v4.log` final `overall.auc = 0.785420`
     - 基线 `ssim` 在 `workspace/kuairand-rand/test.log` 中已有 `overall.auc = 0.785810`
     - 结论：该路线暂不值得继续做三轮复现。
  2. `shavq_v4` 结构结论：
     - `workspace/kuairand-rand/shavq_v4_eval.log`：`overall.auc = 0.785934`
     - `workspace/kuairand-rand/observatory_shavq_v4_stdout.log` 显示：
       - `s1=64/64`, `s2=64/64`，两级码本健康，不是 dead code 问题。
       - `feat_ratio` 从 `0.344` 下降到约 `0.198`，`logit_ratio` 长期仅 `0.014 -> 0.052`，shared 表征有内容，但 shared logit 话语权偏低。
       - `specific_feature_fluctuation` 与 `innovation` 的 `linear_cka` 长期约 `0.88 ~ 0.95`，而与 `context` 仅约 `0.31 ~ 0.47`，说明最终专属修正基本由 innovation 主导。
       - `context` 分支有效秩长期接近 `1`，对应图 `shavq_v4_eff_rank_series.png` 与 `shavq_v4_context_weighted_singular_values.png` 都显示明显低秩塌缩。
     - 结论：`shavq_v4` 不是无效路线，但要继续做，必须同时解决“shared 不入 logit”和“context 分支塌缩”两个结构问题，改造成本偏高。
  3. `vq_share` 结构结论：
     - `workspace/kuairand-rand/test.log` 中真实 `vq_share` 运行段最佳约 `overall.auc = 0.785973`，多次运行集中在 `0.7853 ~ 0.7860`。
     - `workspace/kuairand-rand/observatory/vq_share/vq_codebook_series.png` 显示 `used_code_ratio = 1.0` 且 code usage entropy 长期约 `3.9 ~ 4.0`，码本利用充分。
     - `vq_feature_var_series.png` 显示 shared feature var 长期约 `0.075`，specific feature var 仅约 `0.006 ~ 0.01`，共享主干稳定且主导。
     - `vq_logit_var_series.png` 显示 shared logit var 长期高于或接近 specific logit var，说明 shared 信息能真正穿透到最终决策。
     - `vq_alignment_series.png` 显示 quantized cosine 持续升高到约 `0.93`，残差范数逐步下降，量化共享已稳定对齐。
     - 结论：`vq_share` 是当前最干净、最稳定、最适合继续做“轻量结构增强”的共享底座。
  4. `ple_balanced_v3` 横向结论：
     - `workspace/kuairand-rand/ple_balanced_v3.log` 单轮 `overall.auc = 0.786080`，是当前单日志里最好的一个显式命名结果。
     - `src/custom_models/ple_balanced_v3/README.md` 显示该模型已经形成：
       - `specific/common/balanced` 三路非塌缩门控
       - `balanced` 分支更低的 domain-identifiability
     - 结论：这是值得保留的备选高分线，但当前缺少和 `shavq/vq_share` 一样完整的 observatory 产物与重复验证，暂不宜直接定为唯一主线。
  5. 路线选择：
     - **停止主推**：`curriculum_ssim`、`ple_ob` 纯 gate sharpening、仅靠 `shavq_v4` 温度微调。
     - **下一条主线**：以 `vq_share` 为底座做“轻量门控残差增强”。
     - **备线**：对 `ple_balanced_v3` 做观测化复核与稳定性确认。
     - **不作为当前首选主线的原因**：`shavq_v4` 的问题已经明确，但需要同时修两处核心结构，复杂度高于 `vq_share` 增量改造，不适合当前优先级。
