# 近年论文与代码场景映射

**发布时间**: 2026-06-01 11:15
**发起者**: Agent 1

## 1. 背景与现状分析
`docs/papers/papers/发展过程/` 已经积累了近年多场景推荐的代表论文，但论文方向是否真的适合本仓库的代码环境，还需要与项目里的实验能力做一次对齐。单纯的“论文新颖”不够，必须落到“代码里能不能做、能不能快做、能不能形成完整实验链路”。

这项任务需要把论文方向和项目代码场景做显式映射，避免后面出现选题很新但工程落不下去的问题。

## 2. 明确的执行目标
- [ ] 基于论文材料，列出 3-5 个近年最值得关注的多场景研究方向。
- [ ] 将这些方向分别映射到本仓库的代码环境，判断是否存在现成实现基础、可改造模块或实验数据入口。
- [ ] 给出每个方向在本项目里的落地难度、预期改动范围和验证成本。
- [ ] 明确哪些方向适合快速做出 baseline，哪些方向更适合后续长期研究。
- [ ] 形成一个“论文方向 x 代码可行性”的简表，供后续选题决策直接使用。

#### 方向 1：动态课程学习 (Dynamic Curriculum Learning) 

| 映射维度 | 详情 |
|---------|------|
| **已有 Baseline** | `src/betterbole/models/msr/ssim/` — SSIM 已在 betterbole 中完整集成，`model=ssim` 即可调用 |
| **参考代码** | `reference-projects/SSIM-main/` — 官方代码库，含 HMoE-SSIM、STAR-SSIM 完整实现 |
| **所需数据集** | KuaiRand (已就绪) / Amazon 多域数据 (已就绪 `amazon.py`) |
| **实验场景** | KuaiRand-1K (`@examples/kuairand-1k/kuairan1k.py`) — 最快验证 |
| **改造范围** | 新增 Loss 模块：场景归属度衰减函数 + 课程难度排序；不改变模型架构 |
| **快速 Baseline** | 🟢 **可当天完成** — SSIM 在 KuaiRand 跑 baseline，然后叠加课程调度 |
| **后续长期研究** | 可扩展为多阶段课程、场景自适应难度度量 |
| **落地难度** | ⭐ — 纯 Loss 级改动，SSIM 代码结构清晰 |

#### 方向 2：因果场景效应解耦

| 映射维度 | 详情 |
|---------|------|
| **已有 Baseline** | `src/betterbole/models/msr/sharedbottom.py` — SharedBottom 可作为因果解耦的骨架 |
| **参考代码** | 无直接因果 MSR 代码库；需参考 `CausalInt` 论文实现 |
| **所需数据集** | KuaiRand (已就绪) / AliCCP (已就绪 `aliccp.py`) |
| **实验场景** | KuaiRand-1K 初步验证 + 全量 KuaiRand 正式实验 |
| **改造范围** | 新增因果解耦层（场景 Embedding → 干预变量）；反事实推理模块；梯度正交正则化 |

| **后续长期研究** | 可扩展为完整 Do-Calculus 框架、结构因果模型 |
| **落地难度** | ⭐⭐⭐ — 需要因果推理理论基础，但实验框架成熟 |

#### 方向 3：Pareto/Nash 多场景博弈优化

| 映射维度 | 详情 |
|---------|------|
| **已有 Baseline** | `src/betterbole/models/msr/pareto/` — Pareto 模型已集成！可直接 `model=pareto` |
| **参考代码** | 无专门的 Nash Bargaining 推荐代码库；Improved Accuracy, Declining Orders 论文为分析工作无代码 |
| **所需数据集** | KuaiRand (已就绪) |
| **实验场景** | KuaiRand-1K — Pareto 基线跑通后叠加 Nash Bargaining Loss |
| **改造范围** | 新增 Loss 模块：场景间博弈均衡约束；与现有 Pareto 模型组合 |
| **快速 Baseline** | 🟢 **可 2-3 天完成** — Pareto 模型已在 betterbole 中，直接扩展 Loss |
| **后续长期研究** | 可扩展为多场景 Nash Bargaining 框架，可调节 Trade-off |
| **落地难度** | ⭐⭐ — Loss 层改动，数学框架清晰 |

#### 方向 4：场景概念漂移在线适应

| 映射维度 | 详情 |
|---------|------|
| **已有 Baseline** | 无直接实现；`src/betterbole/core/train/hooks.py` 可作为 Hook 机制接入点 |
| **参考代码** | 无 MSR + Concept Drift 的开源代码 |
| **所需数据集** | KuaiRand + 时间戳切片模拟漂移（需要数据预处理改造） |
| **实验场景** | 需改造 KuaiRand 脚本，添加时间窗划分逻辑 |
| **改造范围** | 新增漂移检测模块 + 在线参数更新机制；BaseTrainer hook 扩展 |
| **快速 Baseline** | 🟡 **需 1-2 周** — 需要先造含时序标签的数据集，再搭建实验协议 |
| **后续长期研究** | 可扩展为完整的场景生命周期管理系统 |
| **落地难度** | ⭐⭐⭐ — 需要设计时序实验协议，数据预处理工作量 |

#### 方向 5：Diffusion 跨场景数据增强

| 映射维度 | 详情 |
|---------|------|
| **已有 Baseline** | 无；betterbole 中没有生成模型模块 |
| **参考代码** | `reference-projects/Diff-MSR-main/` — Diff-MSR 官方代码库可用 |
| **所需数据集** | Amazon / AliCCP 多域（大场景→小场景数据 Bridge）|
| **实验场景** | 独立于现有 betterbole 训练流程——需要加载 Diffusion 生成的数据 |
| **改造范围** | 大：新增 Diffusion 训练 + 条件生成管线；数据增强注入现有 Trainer |
| **快速 Baseline** | 🔴 **需 2-4 周** — Diffusion 模型训练时间长，且生成质量需验证 |
| **后续长期研究** | 可扩展为场景条件生成器、推理时在线增强 |
| **落地难度** | ⭐⭐⭐⭐ — 需要 Diffusion 模型能力，框架改动用大 |

### 关键结论
1. **方向 1（动态课程学习）是唯一满足"当天出 baseline + 高创新性 + 低资源需求"的方向** — SSIM 已集成、KuaiRand 已就绪、改造范围最小
2. **方向 3（Pareto/Nash）是最佳备选** — Pareto 模型已存在，只需扩展 Loss
3. 方向 2（因果）和方向 4（漂移）更适合作为后续顶会冲刺方向
4. 方向 5（Diffusion）工程量大，建议暂缓
