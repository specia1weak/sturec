### 推荐方向（按优先级排序）

---

#### 方向 1（最推荐）：MSR + 动态课程学习 (Dynamic Curriculum Learning)

| 维度 | 评估 |
|------|------|
| **创新性** | ⭐⭐⭐⭐⭐ — SSIM(KDD'25) 证明了"选样本有用"，但它是静态二值选择。动态课程 + 场景归属度软分配是自然且未被探索的延伸 |
| **实验可行性** | ⭐⭐⭐⭐⭐ — Loss 级改动，无需架构变化，基于公开数据集即可完成实验闭环 |
| **资源需求** | ⭐ — 单卡 GPU 即可，无工业数据依赖 |
| **预期风险** | ⭐ — 低风险，课程学习框架在 CV/NLP 中非常成熟 |

**核心问题**：SSIM 静态地选择"高质量共享样本"广播到所有场景，但训练不同阶段需要的"共享程度"不同——初期应鼓励跨场景共享，后期应收敛到场景专有。

**可落地的实验方案**：
- 设计场景归属度衰减函数（训练步数 → 场景隶属权重）
- 基于样本对各场景的"学习难度"做动态课程排序（由易到难）
- 在 Amazon/BookCrossing 等多场景数据集上验证

**最小可行 Baseline**：SSIM (KDD'25) + SharedBottom

**与已有论文的差异点**：SSIM 是静态二值选择（高质量/非高质量），我们是动态连续衰减 + 课程排序

**为什么适合实验室**：纯 Loss/训练法改动，无工程依赖，论文切入点清晰（SSIM 的 natural extension）

---

#### 方向 2（强推荐）：MSR + 因果场景效应解耦 (Causal Scenario Effect Disentanglement)

| 维度 | 评估 |
|------|------|
| **创新性** | ⭐⭐⭐⭐⭐ — CausalInt(KDD'22) 之后 MSR 领域无人深入做过因果方法。CDC(KDD'25) 用因果做域聚类但没做场景效应解耦 |
| **实验可行性** | ⭐⭐⭐ — 需要因果推理基础，但实验可集成到现有框架 |
| **资源需求** | ⭐⭐ — 单卡 GPU 可完成，需要实现因果图和反事实推理模块 |
| **预期风险** | ⭐⭐⭐ — 中风险，因果方法的"故事"需要精心构建 |

**核心问题**：现有 MSR 学到的是场景相关性而非因果效应。例如「用户在支付页购买率高」可能是因为到达支付页的用户本身就有购买意图（选择偏差），而非支付页促进了购买。

**可落地的实验方案**：
- 场景建模为干预变量 → 解耦场景因果效应 vs 用户特征相关性
- 反事实推理：假设用户在场景 B，推荐结果应该是什么？
- 用梯度正交正则化解耦场景相关和用户相关表示

**最小可行 Baseline**：CausalInt (KDD'22) + SharedBottom

**与已有论文的差异点**：CausalInt 做梯度正交正则，我们做显式因果结构建模 + 反事实推理；CDC(KDD'25) 将场景视为聚类变量而非干预变量

**为什么适合实验室**：纯学术创新，不需要工业数据，因果推理在推荐领域关注度高

---

#### 方向 3（推荐）：多场景恶性竞争的 Pareto / Nash 优化

| 维度 | 评估 |
|------|------|
| **创新性** | ⭐⭐⭐⭐ — Improved Accuracy, Declining Orders(KDD'25) 发现了问题但没给解法。将多场景训练建模为博弈论问题是自然且未被探索的 |
| **实验可行性** | ⭐⭐⭐⭐⭐ — Loss 级改动，数学框架清晰（Nash Bargaining / Pareto Frontier） |
| **资源需求** | ⭐ — 单卡 GPU，无需额外数据 |
| **预期风险** | ⭐⭐ — 低风险，博弈论+推荐已有成功先例 |

**核心问题**：Improved Accuracy, Declining Orders(KDD'25) 发现多域联合训练时"精度上升但转化下降"的恶性竞争陷阱，但未给出解法。

**可落地的实验方案**：
- 将多场景联合训练建模为多玩家博弈
- Nash Bargaining 求解均衡点，理论上保证 Pareto 最优
- Loss 层面设计场景间 Trade-off 控制参数

**最小可行 Baseline**：Improved Accuracy, Declining Orders (KDD'25) + Loss Harmonizing (CIKM'23)

**与已有论文的差异点**：已有 Loss 加权是启发式的，我们给出博弈论形式的理论保证

**为什么适合实验室**：纯数学 + Loss 改动，实验周期短，理论贡献清晰

---

#### 方向 4（推荐）：MSR + 场景概念漂移在线适应 (Scenario Concept Drift)

| 维度 | 评估 |
|------|------|
| **创新性** | ⭐⭐⭐⭐ — 所有 MSR 论文假设场景静态。IncMSR(CIKM'24) 做了新增场景适应，但没做场景内概念漂移 |
| **实验可行性** | ⭐⭐⭐⭐ — Online Learning 框架成熟，MSR 领域无人做过 |
| **资源需求** | ⭐⭐ — 需要模拟时序数据的实验设置，单卡 GPU 可用 |
| **预期风险** | ⭐⭐ — 低风险，需要设计时序实验协议 |

**核心问题**：现实场景不是静态的——电商的"大促期"和"平销期"是同一场景但分布完全不同。现有 MSR 无法处理这种场景内的分布变化。

**可落地的实验方案**：
- 场景漂移检测：KL 散度 / MMD 在线监控场景分布
- 检测到漂移后快速调整场景参数（momentum-based update）
- 在添加了时间戳的多场景数据集上划分时间窗验证

**最小可行 Baseline**：IncMSR (CIKM'24) + ADL (CIKM'23)

**与已有论文的差异点**：IncMSR 解决"新增场景"（空间维度），我们解决"场景随时间变化"（时间维度）

**为什么适合实验室**：需要设计时序实验但不需要大规模数据，切入点新颖（MSR+时序=未探索交叉）

---

#### 方向 5（中推荐）：MSR + Diffusion 跨场景数据增强

| 维度 | 评估 |
|------|------|
| **创新性** | ⭐⭐⭐⭐ — Diff-MSR(RecSys'24) 是唯一一篇，被原分析报告评价为"粗糙" |
| **实验可行性** | ⭐⭐⭐ — 需要 Diffusion 模型实现能力 |
| **资源需求** | ⭐⭐⭐ — 需要训练 Diffusion 模型，但推荐特征维度远低于图像，单卡可训练 |
| **预期风险** | ⭐⭐⭐ — 中风险，Diffusion 在推荐中的有效性验证需要实验证明 |

**核心问题**：小/冷启场景数据稀疏。Diff-MSR 做了 Diffusion 增强但只是简单用 Diffusion 生成特征，没有利用场景条件信息。

**可落地的实验方案**：
- 在大场景数据上训练 Diffusion 模型
- 以场景 Embedding 为条件，生成"大场景用户如果在小场景会如何行为"的 Bridge 数据
- 在 Amazon 多域数据集上验证（源域→目标域）

**最小可行 Baseline**：Diff-MSR (RecSys'24) + 场景条件 Conditioning

**与已有论文的差异点**：Diff-MSR 无场景条件生成，我们做场景条件化的跨域数据 Bridge

**为什么适合实验室**：Diffusion 是热门技术，在推荐中的应用还在早期阶段
