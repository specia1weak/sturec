# VQShare 轻量门控残差改造与首轮实验

**发布时间**: 2026-06-01 13:30
**发起者**: Agent 1

## 1. 背景与现状分析
`vq_share` 当前在 KuaiRand-1K 上表现出两点很重要的结构优势：

- 共享量化主干稳定，码本利用充分，shared 特征与 shared logit 都有真实贡献。
- 整体结构简单，比 `shavq_v4` 更适合作为下一步的“低风险增量改造底座”。

但它也有明显上限：

- specific 路较弱，很多运行停在 `0.7857 ~ 0.7860` 一带。
- 目前 `shared + specific` 是固定相加，缺少对“何时相信共享、何时放大修正”的显式控制。

因此下一步不建议直接跳回更复杂的 `shavq` 双路体系，而是先在 `vq_share` 上做一层轻量门控残差增强，验证是否能在不破坏共享稳定性的前提下继续抬升 AUC。

## 3. 执行报告 (移入 3_Done 时填写)
- **执行过程**:
  1. 在 `src/custom_models/vq_share_gate_v1/` 新建轻量门控残差增强模型，保留 `vq_share` 的量化主干与 observatory 产物。
  2. 在 `src/custom_models/__init__.py` 注册 `vq_share_gate_v1`，确保 `@examples/kuairand-1k/kuairan1k.py --model vq_share_gate_v1` 可直接调用。
  3. 用 `seed=2024` 做 smoke test，再补跑 `seed=2024/2025/2026` 的正式验证，其中 2026 结果按用户要求收口到 `epoch2`。
  4. 使用 `workspace/kuairand-rand/observatory/vq_share_gate_v1/*` 和日志逐步分析门控、logit 贡献和 domain 级别收益/损失。
- **结果验证**:
  1. 结构与健康度：
     - `used_code_ratio=1.0`，码本保持完整利用，没有引入 codebook 退化。
     - `vq_gate_series.png` 显示 gate 逐步偏向 `specific`，后期大致到 `gate_specific_mean ~ 0.8`，不是均匀软平均。
     - `vq_gate_scale_series.png` 显示 `shared_scale` 稳定在 `~0.90` 左右，`specific_scale` 稳定在 `~1.13-1.15`，`residual_gain` 大致收敛到 `~0.75-0.78`。
     - `vq_gate_logit_var_series.png` 显示 shared / specific logits 都保持有效方差，整体没有塌缩。
  2. 效果验证：
     - `seed=2024`:
       - epoch1 `overall.auc=0.785794`
       - epoch2 `overall.auc=0.786492`，达到该 seed 的最好点
       - epoch3 `overall.auc=0.786248`
     - `seed=2025`:
       - epoch1 `overall.auc=0.785866`
       - epoch2 `overall.auc=0.786620`，三次里当前最高
       - epoch3 `overall.auc=0.785345`
     - `seed=2026`:
       - epoch1 `overall.auc=0.785621`
       - epoch2 `overall.auc=0.786210`
     - 结论：这条线相对 plain `vq_share` 有稳定增益，best 点已经超过 `ple_balanced_v3` 的 `0.786080`，但整体仍有明显 epoch 后段回落。
  3. 分场景验证：
     - 大场景（对应 `domain0/domain1/domain3`）在 epoch1 -> epoch2 基本持续提升。
     - 小场景 `domain2/domain4` 出现持续掉点，其中 `domain4` 最敏感。
     - `domain2` 对应约 `40.2w` 训练样本，`domain4` 对应约 `18.3w` 训练样本，都是小场景。
     - 这说明本轮收益主要来自对主场景更强的 logit 路由，而不是整体更均衡的场景适配。
  4. 结构归因：
     - 当前收益不是来自 codebook 变差或 shared 失效，而是来自更有效的 logit 侧融合。
     - 但 gate 最终偏向 `specific`，且 `domain1` 的 code usage 明显更强，`domain2/domain4` 仍然偏弱。
     - 因此下一步不应该继续粗暴放大 specific，而应转向“小场景 shared 保护 / domain-aware 再平衡”。
