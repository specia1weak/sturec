## 5. 结论

1. 当前最有效的杠杆不是“借用频率”，而是“外域借用码本是否继续吃当前样本的 VQ 拉力”。
2. `foreign_vq_weight` 从 `0.10` 压到 `0.00` 后，`domain4(tab6)` 明显回升到 `0.7465~0.7477`，这是目前最清晰的正向信号。
3. 单独把 `self_share_prob` 从 `0.90` 提到 `0.95` 只带来小幅波动，不改变主趋势，说明小场景受损的核心问题不是“借得太频繁”，而是“借来之后被强更新”。
4. 现阶段更像是“共享前向借用可以保留，但 foreign codebook 不该承担明显额外 VQ 对齐压力”。

## 6. 后续候选方向

1. 在 `foreign_vq_weight = 0` 的前提下，引入“target residual 与 borrow residual 去相似”约束，增强不同码本路径的区分度。
2. 继续探索两层码本：
   - 第一层共享码本
   - 第二层场景专属码本
   - 剩余残差交给专家网络
3. 评估更大码本是否能提升 KuaiRand-1K 这种高交互量数据的表达能力，但当前优先级低于“更新方向是否正确”。
4. 后续可考虑引入类似 PEPNet 的底层特征差异化增强，但当前阶段先保持底层表征不变，避免变量过多。
