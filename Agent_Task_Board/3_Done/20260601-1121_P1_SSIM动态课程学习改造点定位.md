### 结果验证：SSIM 实现分析

#### 当前 SSIM 架构总览

```
betterbole SSIMModel 结构：
├── shared_embedding (OmniEmbLayer)        ← 主特征嵌入
├── mask_omni_embedding (OmniEmbLayer)     ← 掩码网络专用嵌入
├── domain_hypernet (MLP)                  ← 域超网络
├── mask_heads (num_domains × Linear)      ← 每域一个掩码头
├── branches (num_domains × MLP)           ← 每域一个专家分支
├── TwoStageRetrainMixin                   ← 搜索→再训练阶段控制
│   ├── search_phase: 温度退火 + 软门控 + 共享正则化
│   └── retrain_phase: 冻结掩码模块 + 硬门控
```

#### 与官方 SSIM 代码的差异

| 方面 | betterbole SSIM | 官方 SSIM (SSIM3mask) | 影响 |
|------|----------------|----------------------|------|
| 架构模式 | 通用 num_domains，每域一个 branch MLP | 固定 3 域，每域一个 DNN | betterbole 更通用，改造更简单 |
| 共享正则化 | BCE Loss (share_prob → 1) | KL 散度 (smask → ones) | 可替换，不改变核心逻辑 |
| 门控实现 | Soft: sigmoid(temperature·logits) + LBSign; Hard: retrain_topk per-domain | Soft: 同左; Hard: 全局 topk% | 兼容，改造点对齐 |
| 搜索阶段 | 1 epoch，step 级温度退火 | 同左 | 完全一致 |
| 最终预测 | 各 branch 独立，选 own_branch | 各 domain network 独立，选 own | 一致 |

#### 4 个关键改造点

##### 改造点 1：_compute_share_mask → 场景归属度软分配 (model.py:131-159)

**当前做法**：二值硬门控（share=0 或 1），由 sigmoid(温度·logits) 经阈值决定。

**改造方案**：引入**连续场景归属度权重** `affinity_weights` ∈ [0,1]，替代二值 `share_mask`：
- 每个样本对每个场景的归属度由一个轻量 Attention 模块计算
- 归属度向量经温度退火后，作为加权系数参与 loss 计算
- 早期温度高 → 归属度平滑（跨场景共享多），后期温度低 → 归属度尖锐（趋向专有）

**改动范围**：`model.py` 约 30 行，不改变模型结构。

##### 改造点 2：_phase_loss → 课程感知损失 (model.py:193-222)

**当前做法**：
```
main_loss = (1 - share_mask) * own_branch_loss + share_mask * shared_branch_loss
share_reg = BCE(share_prob, ones)  // 鼓励共享
```

**改造方案**：
```
main_loss = affinity_weights * shared_loss + (1 - affinity_weights) * own_loss
curriculum_reg = adaptive_regularizer(affinity_weights, curriculum_step)
```

关键区别：`affinity_weights` 随课程步数动态变化：
- 步数小 → `affinity_weights` 接近 1（鼓励广泛共享）
- 步数大 → `affinity_weights` 接近 0（收敛到场景专有）

**改动范围**：`model.py` 约 20 行。

##### 改造点 3：custom_train_step → 课程调度器 (model.py:244-273)

**当前做法**：仅做温度退火 `temperature *= growth`。

**改造方案**：新增 `CurriculumScheduler` 模块，管理：
1. **共享度衰减**：`sharing_ratio = cosine_decay(step, total_steps, start=0.8, end=0.2)`
2. **场景归属度温度**：控制 `affinity_weights` 的平滑/尖锐程度
3. **课程阶段标志**：early/mid/late 三个阶段可配置不同策略

**改动范围**：新增 `curriculum.py` 约 80 行，`model.py` 约 15 行。

##### 改造点 4：两阶段训练重载 (two_stage.py / model.py)

**当前做法**：TwoStageRetrainMixin 控制搜索→再训练。

**改造方案**：课程学习不需要两阶段分离——课程本身就是端到端的。因此：
- 禁用 TwoStageRetrainMixin 的阶段切换
- 课程调度器在 `custom_train_step` 中端到端控制
- 移除 retrain phase 的权重冻结和掩码硬化

**改动范围**：`model.py` 约 10 行，移除两阶段相关 hook。

#### 新增模块：CurriculumScheduler

```python
class CurriculumScheduler:
    """
    管理课程学习进度和参数调度。

    核心曲线：
    - sharing_ratio: 从 start_ratio 开始，按 schedule_type 衰减到 end_ratio
      - cosine: 平滑衰减 (推荐)
      - linear: 线性衰减
      - step: 阶段性跳变
    - affinity_temperature: 控制归属度 softmax 的温度
      - 高温 → 各场景权重接近均匀
      - 低温 → 只有主场景权重高
    """
    def __init__(self, total_steps, start_ratio=0.8, end_ratio=0.2,
                 schedule_type="cosine", start_temp=5.0, end_temp=1.0):

    def step(self):
        """推进一个 step，更新当前参数"""

    def get_sharing_ratio(self) -> float:
        """当前共享度 [0,1]"""

    def get_affinity_temperature(self) -> float:
        """当前归属度温度"""

    def get_phase(self) -> str:
        """当前阶段: early/mid/late"""
```
