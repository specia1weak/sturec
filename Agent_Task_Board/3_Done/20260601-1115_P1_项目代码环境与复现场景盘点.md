### 优先复现场景排序

#### 🥇 场景 1：KuaiRand-1K (最推荐，作为研究起点)

| 维度 | 评估 |
|------|------|
| **输入数据** | KuaiRand 短视频推荐数据，已有多域标签 (`domain_id`) |
| **核心脚本** | `@examples/kuairand-1k/kuairan1k.py` (198 行，成熟) |
| **可用模型** | 全部 24 个 MSR 模型，通过 `model=xxx` 参数切换 |
| **运行成本** | 低（3 epoch 默认，单卡 GPU，~分钟级） |
| **适合方向** | 多场景推荐、跨域推荐、多任务推荐 |
| **风险点** | 数据量小（1K 采样），结果仅供快速验证；正式实验需切换全量 KuaiRand |
| **SSIM 已集成** | `ssim` 模型已在注册表中，可直接更换 baseline |

**推荐理由**：训练脚本完整、模型切换灵活、运行成本最低，适合作为所有方向的第一验证场。

#### 🥈 场景 2：MovieLens-1M (序列模型验证)

| 维度 | 评估 |
|------|------|
| **输入数据** | MovieLens 评分数据，含用户序列特征 |
| **核心脚本** | `@examples/ml-1m/ml1m-new.py` (375 行，含 Transformer 序列编码) |
| **可用模型** | 自定义序列模型 + MSR 模型 |
| **运行成本** | 低-中（需序列编码，但数据量小） |
| **适合方向** | 统一序列建模 (USR/ADS/ECAT 类)、场景序列交互 |
| **风险点** | MovieLens 无天然域标签，需人工构造场景划分 |

#### 🥉 场景 3：SSIM 官方代码 + 扩展 (论文方向起点)

| 维度 | 评估 |
|------|------|
| **输入数据** | Amazon / BookCrossing 多域数据 |
| **核心脚本** | `reference-projects/SSIM-main/SSIM3trainer.py` |
| **可用模型** | SSIM + HMoE/STAR 骨架 |
| **运行成本** | 中（训练完整 epoch） |
| **适合方向** | 方向 1（动态课程学习）的直接起点 |
| **优势** | 官方代码可直接运行，betterbole 中 SSIM 已集成，改造路径清晰 |

#### 场景 4：MMLRec Benchmark (标准化评估)

| 维度 | 评估 |
|------|------|
| **输入数据** | 多数据集统一处理 |
| **核心脚本** | `reference-projects/MMLRec/main.py` |
| **适合方向** | Benchmark 对比、基线验证 |
| **风险点** | 配置复杂度较高，适合实验后期而非初期 |

#### 场景 5：自定义 PLE/SHAVQ 系列 (已有基础，适合快速踩坑)

| 维度 | 评估 |
|------|------|
| **输入数据** | 与 KuaiRand/MovieLens 一致 |
| **核心脚本** | `src/custom_models/ple_adv_*/` 等 |
| **适合方向** | MoE 架构改进、共享/专有分离 |
| **风险点** | 已有大量实验记录 (324 行 PLE_ADV 日志)，说明这条路已深度探索但 AUC 收益有限 |

### 关键发现
1. **项目框架成熟** — 完整的模型注册、训练、评估、实验管理链路，24 个 MSR 模型可直接调用
2. **SSIM 已内部集成** — `ssim` 模型已注册，是方向 1（动态课程学习）的完美起点
3. **KuaiRand 是最佳实验场** — 低运行成本 + 完善的多域支持 + 全部模型可切换
4. **16 个官方代码库** — 可直接对标复现，验证 betterbole 实现正确性
5. **多个 Loss 级方法已有集成** — Pareto 优化、Crocodile (Cov Loss) 都已实现，方便组合创新
