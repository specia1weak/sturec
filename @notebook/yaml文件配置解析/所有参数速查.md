# 环境设置

环境设置旨在设置运行环境的基本参数。

- **`gpu_id`** (`str`): 可用 GPU 设备的 ID。默认为 `0`。
- **`worker`** (`int`): 处理数据的工作进程数。默认为 `0`。
- **`seed`** (`int`): 随机种子。默认为 `2020`。
- **`state`** (`str`): 日志级别。默认为 `'INFO'`。取值范围为 `['INFO', 'DEBUG', 'WARNING', 'ERROR', 'CRITICAL']`。
- **`encoding`** (`str`): 用于读取原子文件的编码格式。默认为 `'utf-8'`。
- **`reproducibility`** (`bool`): 如果为 `True`，则工具将使用确定性卷积算法，从而使结果可复现。如果为 `False`，则工具将对多种卷积算法进行基准测试并选择最快的算法，这将导致结果不可复现，但在某些情况下可以加快模型训练速度。默认为 `True`。
- **`data_path`** (`str`): 输入数据集的路径。默认为 `'dataset/'`。
- **`checkpoint_dir`** (`str`): 保存检查点（checkpoint）文件的路径。默认为 `'saved/'`。
- **`show_progress`** (`bool`): 是否显示训练和评估轮数（epoch）的进度条。默认为 `True`。
- **`save_dataset`** (`bool`): 是否保存筛选后的数据集。如果为 `True`，则保存筛选后的数据集；否则，不保存。默认为 `False`。
- **`dataset_save_path`** (`str`): 已保存数据集的路径。该工具将尝试从此路径加载数据集。如果此值为 `None`，则该工具将尝试从 `{checkpoint_dir}/{dataset}-{dataset_class_name}.pth` 加载数据集。如果已保存数据集的配置与当前配置不匹配，则该工具将从头开始创建数据集。默认为 `None`。
- **`save_dataloaders`** (`bool`): 是否保存拆分后的数据加载器（Dataloader）。如果为 `True`，则保存拆分后的数据加载器；否则，将不保存。默认为 `False`。
- **`dataloaders_save_path`** (`str`): 已保存数据加载器的路径。该工具将尝试从此路径加载数据加载器。如果此值为 `None`，则该工具将尝试从 `{checkpoint_dir}/{dataset}-for-{model}-dataloader.pth` 加载数据加载器。如果已保存数据加载器的配置与当前配置不匹配，则该工具将从头开始创建数据加载器。默认为 `None`。
- **`log_wandb`** (`bool`): 是否使用 Weights & Biases (W&B)。如果为 `True`，则使用 W&B 可视化不同实验的配置和指标；否则，将不使用。默认为 `False`。
- **`wandb_project`** (`str`): 在 W&B 中进行实验的项目名称。默认为 `'recbole'`。
- **`shuffle`** (`bool`): 是否在每个 epoch 之前对训练数据打乱顺序。默认为 `True`。

# 原子文件格式

- **`field_separator`** (`str`): 原子文件中不同列的分隔符。默认为 `"\t"`。
- **`seq_separator`** (`str`): 序列特征内部的分隔符。默认为 `" "`。

# 基本信息

## 公共特征

- **`USER_ID_FIELD`** (`str`): 用户 ID 特征的字段名称。默认为 `user_id`。
- **`ITEM_ID_FIELD`** (`str`): 物品 ID 特征的字段名称。默认为 `item_id`。
- **`RATING_FIELD`** (`str`): 评分特征的字段名称。默认为 `rating`。
- **`TIME_FIELD`** (`str`): 时间戳特征的字段名称。默认为 `timestamp`。
- **`seq_len`** (`dict`): 键是序列特征的字段名，值是每个序列的最大长度（这意味着过长的序列将被截断）。如果未设置，则不会截断序列。默认为 `None`。

## Pointwise（单样本行数据集） 数据加载器标签

- **`LABEL_FIELD`** (`str`): 生成的标签的预期字段名称。默认为 `label`。
- **`threshold`** (`dict`): 格式为 `{k (str): v (float)}`。将根据 `inter_feat[k]` 和 `v` 的值生成 0/1 标签。`inter_feat` 中 `inter_feat[k] >= v` 的行将被标记为正例，否则标记为负例。请注意，`threshold` 中最多只能存在一对 `k` 和 `v`。默认值为 `None`。

## Pairwise （样本对数据集）数据加载器的负采样前缀

- **`NEG_PREFIX`** (`str`): 作为负例生成的字段名称的前缀。例如，如果我们有一个名为 `item_id` 的正例物品 ID，那么负例样本中的物品 ID 将被命名为 `NEG_PREFIX + item_id`。默认为 `neg_`。

## 序列模型所需

- **`ITEM_LIST_LENGTH_FIELD`** (`str`): 表示物品序列长度的特征字段名称。默认为 `item_length`。
- **`LIST_SUFFIX`** (`str`): 生成为序列的字段名称的后缀。例如，如果我们有一个名为 `item_id` 的物品 ID，那么这些物品 ID 序列将命名为 `item_id + LIST_SUFFIX`。默认为 `_list`。
- **`MAX_ITEM_LIST_LENGTH`** (`int`): 每个生成的序列的最大长度。默认为 `50`。
- **`POSITION_FIELD`** (`str`): 生成的位置序列的字段名称。对于长度为 `k` 的序列，其位置序列为 `range(k)`。请注意，仅当此参数不为 `None` 时才会生成此字段。默认为 `position_id`。

## 知识图谱模型所需

- **`HEAD_ENTITY_ID_FIELD`** (`str`): 头实体 ID 特征的字段名称。默认为 `head_id`。
- **`TAIL_ENTITY_ID_FIELD`** (`str`): 尾实体 ID 特征的字段名称。默认为 `tail_id`。
- **`RELATION_ID_FIELD`** (`str`): 关系 ID 特征的字段名称。默认为 `relation_id`。
- **`ENTITY_ID_FIELD`** (`str`): 实体 ID 的字段名称。请注意，它只是实体的符号，而不是 `xxx_feat` 中某个实体的实际特征。默认为 `entity_id`。
- **`kg_reverse_r`** (`bool`): 是否反转双向边的三元组关系。默认为 `False`。
- **`entity_kg_num_interval`** (`str`): 具有区间格式，例如 `[A,B]` / `[A,B)` / `(A,B)` / `(A,B]`，其中 A 和 B 是区间的端点，且 A <= B。三元组数量在区间内的实体（包括头实体和尾实体）将被保留。默认值为 `[0,inf)`。
- **`relation_kg_num_interval`** (`str`): 具有区间格式，例如 `[A,B]` / `[A,B)` / `(A,B)` / `(A,B]`，其中 A 和 B 是区间的端点，且 A <= B。三元组数量在区间内的关系将被保留。默认值为 `[0,inf)`。

# 选择性加载

- **`load_col`** (`dict`): 键是已加载原子文件的后缀，值是要加载的字段名称列表。如果 `load_col` 中不存在某个后缀，则不会加载相应的原子文件。注意，如果 `load_col` 为 `None`，则会加载所有已存在的原子文件。默认值为 `{inter: [user_id, item_id]}`。
- **`unload_col`** (`dict`): 键是已加载原子文件的后缀，值是不加载的字段名称列表。请注意，`load_col` 和 `unload_col` 不能同时设置。默认值为 `None`。
- **`unused_col`** (`dict`): 键是已加载原子文件的后缀，值是为数据处理加载但模型中不会使用的字段名称列表。例如，`time_field` 可能用于时间排序，但模型不会使用此字段。默认为 `None`。
- **`additional_feat_suffix`** (`list`): 控制加载额外的原子文件。例如，如果要从 `ml-100k.hello` 加载特征，只需将此参数设置为 `additional_feat_suffix: [hello]`。额外特征的特征将存储在 `Dataset.feat_list` 中。默认为 `None`。
- **`numerical_features`** (`list`): 要嵌入到上下文感知方法中的数值特征。默认为 `None`。

# 过滤

## 移除重复的用户-物品交互

- **`rm_dup_inter`** (`str`): 是否删除重复的用户-物品交互。如果 `time_field` 存在，则 `inter_feat` 将按 `time_field` 升序排序。否则，将保持不变。之后，如果 `rm_dup_inter='first'`，则保留重复用户-物品交互中的第一个；如果 `rm_dup_inter='last'`，则保留重复交互中的最后一个。默认值为 `None`。

## 按值筛选

- **`val_interval`** (`dict`): 格式为 `{k (str): interval (str), ...}`，其中 `interval` 可以设置为 `[A,B]` / `[A,B)` / `(A,B)` / `(A,B]`。`feat` 中 `feat[k]` 位于 `interval` 内的行将被保留。如果要指定多个区间，请用分号分隔。例如，可以使用 `{k: "[A,B);(C,D]"}`，`feat[k]` 位于任何指定区间范围内的行都将被保留。默认值为 `None`，表示将保留所有行。

## 移除用户或物品的交互
- **`filter_inter_by_user_or_item`** (`bool`): 如果为 `True`，我们将删除 `inter_feat` 中用户或物品不在 `user_feat` 或 `item_feat` 中的交互。默认为 `True`。
## 按交互次数筛选
- **`user_inter_num_interval`** (`str`): 具有区间格式，例如 `[A,B]` / `[A,B)` / `(A,B)` / `(A,B]`，其中 A 和 B 是区间的端点，A <= B。交互次数在区间内的用户将被保留。默认值为 `[0,inf)`。
- **`item_inter_num_interval`** (`str`): 具有区间格式，例如 `[A,B]` / `[A,B)` / `(A,B)` / `(A,B]`，其中 A 和 B 是区间的端点，A <= B。交互次数在区间内的物品将被保留。默认值为 `[0,inf)`。

# 预处理

- **`alias_of_user_id`** (`list`): 字段名称列表，这些字段名称将与 `USER_ID_FIELD` 重新映射到同一个索引系统中。默认为 `None`。
- **`alias_of_item_id`** (`list`): 字段名称列表，这些字段名称将与 `ITEM_ID_FIELD` 重新映射到同一索引系统中。默认为 `None`。
- **`alias_of_entity_id`** (`list`): 字段名称列表，这些字段名称将与 `ENTITY_ID_FIELD`、`HEAD_ENTITY_ID_FIELD` 和 `TAIL_ENTITY_ID_FIELD` 重新映射到同一个索引系统中。默认为 `None`。
- **`alias_of_relation_id`** (`list`): 字段名称列表，这些字段名称将与 `RELATION_ID_FIELD` 重新映射到同一索引系统中。默认为 `None`。
- **`preload_weight`** (`dict`): 格式为 `{k (str): v (float), ...}`。`k` 是一个标记字段（token），表示预加载权重矩阵每一行的 ID；`v` 是一个浮点类型的字段。每对 `k` 和 `v` 必须来自同一个原子文件。此参数可用于加载预训练向量。默认为 `None`。
- **`normalize_field`** (`list`): 要归一化的字段名称列表。请注意，只有浮点型字段才能归一化。默认为 `None`。
- **`normalize_all`** (`bool`): 如果为 `True`，则对所有浮点型字段进行归一化。默认为 `None`。
- **`discretization`** (`dict`): 格式为 `{k (str): v (dict), ...}`。`k` 是一个浮点数字段，表示要离散化的字段。`v` 是一个配置字典，包含两个键：`method` 和 `bucket`，分别控制离散化策略。
  - **`method`** (`str`): 决定浮点数据的离散化方式。目前支持两种离散化策略：`['ED', 'LD']`。如果值为 `ED`，则数据将按等距离散化。如果值为 `LD`，则数据将按对数离散化。默认值为 `ED`。
  - **`bucket`** (`int`): 当离散化方法为 `ED` 时，包含相同数量特征的桶的数量。

# 基准测试文件

- **`benchmark_filename`** (`list`): 预先分割的用户-物品交互后缀列表。我们只会应用归一化和重映射 ID，不会删除 `inter_feat` 中的交互记录。然后根据 `benchmark_filename` 分割 `inter_feat`。例如，假设数据集名为 `click`，`benchmark_filename` 等于 `['part1', 'part2', 'part3']`。那么我们将加载 `click.part1.inter`、`click.part2.inter` 和 `click.part3.inter`，并将它们分别作为训练集、验证集和测试集。默认值为 `None`。

# 训练设置

训练设置旨在设定模型训练相关的参数。

- **`epochs`** (`int`): 训练的轮数。默认为 `300`。
- **`train_batch_size`** (`int`): 训练的批次大小。默认为 `2048`。
- **`learner`** (`str`): 所使用的优化器名称。默认为 `'adam'`。取值范围为 `['adam', 'sgd', 'adagrad', 'rmsprop', 'sparse_adam']`。
- **`learning_rate`** (`float`): 学习率。默认为 `0.001`。
- **`train_neg_sample_args`** (`dict`): 训练负采样参数。此参数包含 4 个键：`distribution`、`sample_num`、`dynamic` 和 `candidate_num`。
  - **`distribution`** (`str`): 决定采样池中负样本的分布方式。目前支持两种分布：`['uniform', 'popularity']`。`uniform` 表示均匀地选择负样本，而 `popularity` 表示根据物品的流行度（`.inter` 文件中的 `Counter(item)`）来选择负样本。默认值为 `uniform`。
  - **`sample_num`** (`int`): 决定我们要采样的负样本数量。默认值为 `1`。
  - **`dynamic`** (`bool`): 决定是否采用动态负采样。默认值为 `False`。
  - **`candidate_num`** (`int`): 决定动态负采样时候选负样本的数量。默认值为 `0`。
- **`eval_step`** (`int`): 在验证集上进行评估之前的训练轮数。如果小于 1，则模型不会在验证集上进行评估。默认为 `1`。
- **`stopping_step`** (`int`): 基于验证集的提前停止（early stopping）的阈值步数。默认为 `10`。
- **`clip_grad_norm`** (`dict`): 传递给 `clip_grad_norm_` 的参数，用于裁剪模型的梯度范数。默认为 `None`。
- **`loss_decimal_place`** (`int`): 训练损失保留的小数位数。默认为 `4`。
- **`weight_decay`** (`float`): 用于优化器的权重衰减（L2 惩罚项）。默认为 `0.0`。
- **`require_pow`** (`bool`): 该标志用于确认在 `EmbLoss` 中是否基于范数执行幂运算。默认为 `False`。
- **`enable_amp`** (`bool`): 该参数决定是否使用混合精度训练。默认为 `False`。
- **`enable_scaler`** (`bool`): 该参数决定是否使用 `GradScaler`，它通常与混合精度训练配合使用以避免梯度精度溢出。默认为 `False`。

# 评估设置

评估设置旨在设定模型评估的参数。
- **`eval_args`** (`dict`): 此参数有 4 个键：`group_by`、`order`、`split` 和 `mode`，分别控制模型评估的数据分组策略、数据排序策略、数据分割策略和评估模式。
  - **`group_by`** (`str`): 决定 `.inter` 中数据的分组方式。目前支持两种分组策略：`['user', 'none']`。如果 `group_by` 的值为 `user`，则数据将按 `USER_ID_FIELD` 列分组，并按用户维度拆分。如果值为 `none`，则数据不进行分组。默认值为 `user`。
  - **`order`** (`str`): 决定 `.inter` 文件中数据的排序方式。目前我们支持两种排序策略：`['RO', 'TO']`，分别表示随机排序（Random Ordering）和时间排序（Temporal Ordering）。对于 `RO`，我们会先打乱数据顺序，然后按此顺序拆分数据。对于 `TO`，我们会先按 `TIME_FIELD` 列升序排列数据，然后按此顺序拆分数据。默认值为 `RO`。
  - **`split`** (`dict`): 决定如何分割 `.inter` 中的数据。目前我们支持两种分割策略：`['RS', 'LS']`，分别表示基于比例的数据分割（Ratio Split）和留一法数据分割（Leave-one-out Split）。如果 `split` 的键是 `RS`，则需要设置分割比例，例如 `[0.8,0.1,0.1]`、`[7,2,1]` 或 `[8,0,2]`，分别表示训练集、验证集和测试集的比例。如果 `split` 的键是 `LS`，则目前我们支持三种 `LS` 模式：`['valid_and_test', 'valid_only', 'test_only']`，需要选择其中一种模式作为 `LS` 的值。默认值为 `{'RS': [0.8,0.1,0.1]}`。
  - **`mode`** (`str|dict`): 决定在验证（valid）和测试（test）阶段评估模型时的数据范围。目前我们支持四种评估模式：`['full', 'unixxx', 'popxxx', 'labeled']`。`full`、`unixxx` 和 `popxxx` 专为隐式反馈（无标签数据）评估而设计。对于隐式反馈，我们将观察到的交互视为正样本，将未观察到的交互视为负样本。
    - `full` 表示在所有物品的集合上评估模型。
    - `unixxx`（例如 `uni100`）表示为测试集中的每个正样本均匀抽取 100 个负样本，并使用这些正样本及其抽取的负样本来评估模型。
    - `popxxx`（例如 `pop100`）表示基于物品的流行度（`.inter` 文件中的 `Counter(item)`）为测试集中的每个正样本抽取 100 个负样本，并使用这些正样本及其抽取的负样本来评估模型。这里的 `xxx` 必须为整数。
    - 对于显式反馈（带标签的数据），您应该将 `mode` 设置为 `labeled`，我们将根据您的标签评估模型。
    - 您可以使用 `valid` 和 `test` 作为字典键，在不同阶段设置特定的 `mode`。默认值为 `full`，相当于 `{'valid': 'full', 'test': 'full'}`。
- **`repeatable`** (`bool`): 是否使用可重复的推荐场景评估结果。请注意，对于序列模型，此选项已禁用，因为序列推荐本身就是可重复的。对于其他模型，默认值为 `False`。
- **`metrics`** (`list` or `str`): 评估指标。默认为 `['Recall', 'MRR', 'NDCG', 'Hit', 'Precision']`。可选范围如下表所示：
  
- | 类型 | 指标 |
  | --- | --- |
  | **基于排名** | 召回率 (Recall)、MRR、NDCG、命中率 (Hit)、MAP、精确率 (Precision)、GAUC、项目覆盖率 (Item Coverage)、平均受欢迎程度 (Average Popularity)、基尼系数 (Gini Index)、香农熵 (Shannon Entropy)、尾部百分比 (Tail Percentage) |
  | **基于价值** | AUC、MAE、RMSE、LogLoss |

  *请注意，基于价值的指标和基于排名的指标不能同时使用。*
- **`topk`** (`list` or `int` or `None`): Top-K 评估指标的 K 值。默认为 `10`。
- **`valid_metric`** (`str`): 用于提前停止（Early Stopping）的评估指标。它必须是 `metrics` 中已使用的指标之一。默认为 `'MRR@10'`。
- **`eval_batch_size`** (`int`): 评估时的批次大小（Batch Size）。默认为 `4096`。
- **`metric_decimal_place`** (`int`): 指标分数的保留小数位数。默认为 `4`。


## 额外参数
recbole的内置模型也有各自额外的参数，在`recbole/properties/model/`下

例如DIN
```yaml
embedding_size: 10              # (int) The embedding size of features.
mlp_hidden_size: [256,256,256]  # (list of int) The hidden size of MLP layers. 
dropout_prob: 0.0               # (float) The dropout rate.                 
pooling_mode: 'mean'            # (str) Pooling mode of sequence data.
```
